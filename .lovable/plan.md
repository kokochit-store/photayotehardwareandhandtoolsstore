
# Project Audit Report

Read-only review across security, UI/UX, and performance/code quality. No code was modified. Findings are grouped by severity; each item lists the location and a recommended fix.

Scanner status: latest `supabase_lov`, `supabase`, and `supply_chain` scans all return zero persisted findings, so this report focuses on issues the scanners do not catch (app-level logic, UX, and code quality).

---

## Critical

**C1. Admin edge function accepts caller as admin without `has_role` gate** — `supabase/functions/generate-product-image/index.ts`
Admin check queries `user_roles` directly with the service-role client and treats any row as authorization. This works today, but the correct pattern is to call the SECURITY DEFINER `public.has_role(_user_id, 'admin')` RPC (already defined in the DB) to keep the authorization logic in one auditable place and immune to future schema drift.
Fix: replace the direct `.from("user_roles")` select with `admin.rpc("has_role", { _user_id: userData.user.id, _role: "admin" })` and reject on `!data`.

**C2. `products.id` assigned by client with a read-then-write race** — `src/pages/Admin.tsx` (`addProduct`, lines ~125–142)
`nextId = max(id)+1` computed in the browser. Two concurrent admins (or a double-click) can pick the same id and one insert will fail or, worse, overwrite. Also, this is only enforced by the PK, not by any sequence.
Fix: add a Postgres identity/sequence to `products.id` (`ALTER TABLE products ALTER COLUMN id ADD GENERATED BY DEFAULT AS IDENTITY`) and stop sending `id` from the client.

## High

**H1. Storage bucket `product-images` is public and writable by admins with no MIME/size enforcement server-side**
Admin upload path (`Admin.tsx` `handleUpload`) does client-side size cap only (20 MB) and content-type is whatever the browser reports. A compromised admin token or a bug could upload arbitrary files (including HTML/SVG) served from a public bucket, enabling stored-XSS via the CDN origin.
Fix: on the bucket set `allowed_mime_types` (`image/jpeg`, `image/png`, `image/webp`) and a `file_size_limit`; keep upload paths under `products/`.

**H2. No CSP / security headers on the SPA**
`index.html` has no `Content-Security-Policy`, `Referrer-Policy`, `X-Content-Type-Options`, or `Permissions-Policy`. Meaningful defence against XSS/clickjacking is missing. This is especially relevant because product `image_url` is rendered directly from the DB.
Fix: add `<meta http-equiv="Content-Security-Policy" content="…">` limiting `img-src` to `self`, the Supabase storage host, and `data:`; add `Referrer-Policy: strict-origin-when-cross-origin` and `X-Content-Type-Options: nosniff` via meta tags or hosting config.

**H3. CSV/JSON import trusts arbitrary user files** — `src/components/ProductImportExport.tsx`
Fields go straight into `insert`/`update` with no length caps, no numeric sanity limits, and no rejection of extra keys. A large file (100k rows) is processed on the main thread and can hang the tab; a malicious `image_url` (`javascript:` or `data:text/html`) will be stored and later rendered.
Fix: validate each row with a zod schema (bounded string lengths, non-negative numbers, `image_url` restricted to `https://` and to the storage host), cap total rows (e.g. 5,000), and reject anything else with a clear error.

**H4. Search input built via string interpolation into `.or()` filter** — `src/hooks/useProducts.ts` line ~31 and `src/pages/Admin.tsx` line ~163
`` `name.ilike.%${term}%,sku.ilike.%${term}%` `` — a term containing `,`, `)`, or `\` can break the PostgREST filter grammar and, in worst case, alter the query intent. Not classic SQL injection (PostgREST does its own parsing) but still user input assembling a filter DSL.
Fix: escape or reject `%`, `,`, `\`, `(`, `)`, and `*` in `term`; or split into two queries and merge, or use `.textSearch`.

**H5. AuthContext calls Supabase from inside `onAuthStateChange`**
`AuthContext.tsx` wraps the role fetch in `setTimeout(…, 0)` (correct) but the initial `getSession().then(async …)` also awaits a Supabase call before flipping `loading=false`. If the token is stale/expired, the whole app is stuck on the loader. Also, there is no error path — a failed role lookup silently leaves `isAdmin=false` with no toast.
Fix: catch errors, set `loading=false` in `finally`, and surface a toast on unexpected failures.

## Medium

**M1. `ProductCard` `alt` text is the raw product name; decorative badges have no `aria-label`s tuned for screen readers** — `src/components/ProductCard.tsx`
`Star` icons convey the rating visually only. The numeric `(4.7)` beside them is announced, but the star row itself is not marked `aria-hidden`, so screen readers can announce five "img" nodes.
Fix: wrap star row in `<div aria-hidden="true">` and add `aria-label={`Rated ${rating} out of 5`}` on the container.

**M2. Icon-only buttons without accessible names**
`CartDrawer.tsx` close/quantity buttons (`X`, `Minus`, `Plus`, `Trash2`), `Navbar.tsx` desktop "Search" text button is fine but the mobile search input inside `mobileSearchOpen` has no `aria-label`, and `ProductCard`'s "Add" hover button relies on hover — keyboard/touch users cannot see it. All fail WCAG `button-name` or discoverability.
Fix: add `aria-label` to each icon button; make the Add-to-cart button always visible on touch viewports (e.g. `md:opacity-0 md:group-hover:opacity-100` instead of the current always-hover class).

**M3. Search input in Navbar is decorative** — `Navbar.tsx` lines 54–63
The desktop and mobile search inputs are wired to nothing. Users typing and hitting Enter get no result. This is a broken affordance, not a real feature.
Fix: on submit, `navigate(`/products?search=${encodeURIComponent(value)}`)` and have `Products.tsx` read `search` from the URL.

**M4. Cart is in-memory only**
`CartContext.tsx` never persists to `localStorage`. A page reload empties the cart mid-checkout. There is also no `Checkout` handler wired up (`CartDrawer.tsx` line 91) — the primary CTA does nothing.
Fix: hydrate/persist `items` to `localStorage` (guarded by `try/catch`) and either implement Checkout or clearly disable/label the button.

**M5. Rate-limited role check on every auth state change**
`AuthContext.tsx` re-queries `user_roles` on every auth event (token refresh included, which Supabase fires every hour). Cache the result for the session or invalidate only on `SIGNED_IN`/`SIGNED_OUT`.

**M6. `LiveVisitors` (assumed simulated) and `ReorderList` fetch on mount without a "stale" indicator**
`ReorderList.tsx` refetches only via manual button; a threshold change does not re-fetch until Refresh is clicked (the `useEffect` deps list is disabled via eslint-comment).
Fix: include `threshold` in the effect deps (debounced) so the list stays in sync, or disable the numeric input until Refresh.

**M7. Hero image not preloaded; LCP suffers**
`HeroSection.tsx` uses `/images/hero-bg.jpeg` via `background-image`. Add `<link rel="preload" as="image" href="/images/hero-bg.jpeg" fetchpriority="high">` in `index.html`, and serve a WebP/AVIF variant.

**M8. Products query has no index awareness / uses `count: "exact"` on every page fetch**
`useProducts.ts` requests `count: "exact"` even during pagination navigation, forcing a full count. Switch to `estimated` after the first page for large catalogs.

## Low

- **L1. `formatPrice` inconsistency.** `ProductCard`, `CartDrawer`, and `Admin` each define their own (`en-US`, `my-MM`, plain). Memory rule requires Arabic numerals + "ကျပ်". `CartDrawer.tsx` uses `my-MM` which renders Myanmar numerals — violates the pricing-display memory. Standardize on one helper in `src/lib/utils.ts`.
- **L2. `<html lang="en">` in `index.html`** but the UI is mostly Burmese. Set `lang="my"` (or per-page via helmet) so screen readers pick the correct voice.
- **L3. Missing single `<main>` landmark.** `Index.tsx`, `Products.tsx`, `Admin.tsx` each render `<main>`, which is fine; `Auth.tsx` also does — good. But `App.tsx` renders `Navbar` + `CartDrawer` as siblings to `<main>` inside the route — verify no page nests a `<main>` inside another.
- **L4. Filter button on `/products` is inert** (`Products.tsx` line 87). Remove or wire up.
- **L5. `Products` page uses `searchParams` in a `useEffect` dep array without stable ref**, causing extra renders. Read once via `searchParams.get()` and add `searchParams.toString()` to deps if you need reactivity.
- **L6. `Admin.tsx` uses a native `<select>`** (line 406) among shadcn components — inconsistent styling; use `Select` from `@/components/ui/select`.
- **L7. Console errors from edge function are logged with `console.error("generate-product-image error:", err)`** — includes the raw error message which may contain third-party payloads. Prefer a truncated, structured log.
- **L8. `queryClient` is created at module scope** — fine, but no default `staleTime`/`retry` → every navigation refetches. Configure `defaultOptions` on `QueryClient`.
- **L9. `og:image` absent.** Hosting injects it, per Lovable rule — leave as is, do not hardcode.
- **L10. No 404 SEO tags** on `NotFound.tsx` (verify separately).
- **L11. Rating is hardcoded to 4.7** in `ProductCard` — either hide the stars until real reviews exist, or add a `reviews` table.

4. H5, M3, M4 — user-visible reliability (auth spinner, working search, persistent cart).
5. M-series a11y and perf.
6. L-series polish.
